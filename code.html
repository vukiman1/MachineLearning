An introduction to the core concepts of Machine Learning.

A brief, engaging paragraph defining Machine Learning in simple terms. It explains how computers can learn from data to make predictions or decisions without being explicitly programmed for the task.

ML vs. AI, DL, Data Mining

This section clarifies the distinctions and overlaps between these related fields. Artificial Intelligence (AI) is the broad concept of machines being able to carry out tasks in a way that we would consider "smart." Machine Learning (ML) is a subset of AI that includes the statistical methods enabling machines to improve at tasks with experience. Deep Learning (DL) is a subset of ML which uses multi-layered neural networks to achieve state-of-the-art accuracy.

The Machine Learning Pipeline

The ML pipeline is a sequence of steps to take a project from raw data to a deployed model. It includes data collection, preprocessing, model training, evaluation, and deployment.

Key Terminology

Model

The output of a machine learning algorithm run on data; a mathematical representation of a real-world process.

Features

The individual measurable properties or characteristics of a phenomenon being observed.

Loss Function

A method of evaluating how well your algorithm models your dataset. If your predictions are totally off, your loss function will output a higher number.

Training/Testing Set

The dataset is split into a training set, used to train the model, and a testing set, used to evaluate its performance on unseen data.

Introduction to Hill Climbing

Hill climbing is a mathematical optimization technique which belongs to the family of local search. It is an iterative algorithm that starts with an arbitrary solution to a problem, then attempts to find a better solution by making an incremental change to the solution.

pseudocode 
procedure hillClimbing(problem)
  current_node = problem.initial_state
  loop do
    neighbors = problem.get_neighbors(current_node)
    if neighbors is empty then
      return current_node // Reached a peak
    end if
    
    best_neighbor = find_best(neighbors, problem.objective_function)
    
    if problem.objective_function(best_neighbor) <= problem.objective_function(current_node) then
      return current_node // Current is better or equal, stop
    end if
    
    current_node = best_neighbor
  end loop
end procedure
